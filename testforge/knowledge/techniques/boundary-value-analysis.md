# 边界值分析（Boundary Value Analysis）

## 原理

错误最容易出现在输入范围的边界处。测试边界值本身以及刚好越过边界的值。

## 适用场景

- 数值字段有 min/max 限制（如端口号 0-65535）
- 字符串字段有最大/最小长度（如用户名 max_length=50）
- 数组/列表有长度限制
- 分页参数（page、page_size）

## 应用步骤

1. 识别每个参数的边界：最小值、最大值
2. 为每个边界生成测试值：
   - **恰好在边界上**：min、max
   - **刚好越界**：min-1、max+1
   - **边界内侧**：min+1、max-1
3. 特殊边界：0、空串（长度 0）、1（最小正整数）

## 示例

字段 `systemHost`，max_length=50：
- 长度 0（空串）→ 必填校验失败
- 长度 1 → 通过（最短有效）
- 长度 49 → 通过（最大有效-1）
- 长度 50 → 通过（恰好最大）
- 长度 51 → 失败（超过最大）

字段 `codePage`，range [0, 999999]：
- 值 -1 → 失败
- 值 0 → 通过
- 值 1 → 通过
- 值 999999 → 通过
- 值 1000000 → 失败

## 常见陷阱

- 只测最大不测最小（反之亦然）
- 忽略 0 这个特殊边界
- 字符串长度边界和数值大小边界混淆
- 忘记测试"刚好越界"的值（max+1）
