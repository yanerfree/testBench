# 状态迁移（State Transition）

## 原理

系统在不同状态间转换，特定操作只在特定状态下合法。测试所有合法状态转换和非法状态转换。

## 适用场景

- 资源有生命周期（草稿 → 发布 → 归档 → 删除）
- 操作有前置条件（必须先连通测试，才能保存）
- 流程有顺序依赖（创建 → 编辑 → 启用 → 停用）
- 并发状态变更

## 应用步骤

1. 识别所有状态
2. 识别所有事件/操作
3. 画出状态转换图：
   ```
   [未创建] --创建--> [已创建/未测试]
   [已创建/未测试] --连通测试成功--> [已测试]
   [已测试] --保存--> [已保存]
   [已保存] --编辑--> [已保存]（更新）
   [已保存] --删除--> [已删除]
   ```
4. 生成测试用例：
   - **合法转换**：每条边一个用例
   - **非法转换**：在错误状态下执行操作（如未测试就保存）

## 示例

RFC 连接器：
- 合法：创建 → 连通测试 → 保存 ✓
- 非法：创建 → 直接保存（跳过连通测试）→ 应该拒绝
- 非法：已删除的连接器 → 编辑 → 应该返回 404
- 合法：已保存 → 重新连通测试 → 更新保存 ✓

## 常见陷阱

- 只测试正向（happy path）状态转换
- 忽略"跳过中间步骤"的非法转换
- 忽略并发场景（两个用户同时操作同一资源）
- 忽略已删除/已归档状态下的操作
