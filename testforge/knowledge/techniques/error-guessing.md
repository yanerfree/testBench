# 错误推测（Error Guessing）

## 原理

基于经验和直觉，推测系统可能出错的地方。不依赖系统化的技法，而是依赖测试人员的经验积累和常见错误模式。

## 适用场景

- 上述 5 种技法覆盖不到的"软"场景
- 安全相关测试（注入、越权）
- 性能边界（大数据量、高并发）
- 兼容性和环境差异

## 常见错误模式库

### 输入类
- null / undefined / 空对象 {}
- 空字符串 "" vs 纯空格 "   "
- 超长字符串（超过字段限制 10 倍以上）
- 特殊字符：`<script>alert(1)</script>`、`'; DROP TABLE users; --`
- Unicode：中文、emoji、零宽字符、RTL 字符
- 数值溢出：Integer.MAX_VALUE + 1、负数、小数、科学计数法
- 数组：空数组 []、超大数组、嵌套数组

### 认证授权类
- 无 token 请求
- 过期 token
- 篡改 token（修改 payload）
- 用其他用户的 token 操作非自己的资源
- 低权限用户调用高权限接口

### 并发类
- 同时创建同名资源（唯一性约束竞争）
- 读取正在被修改的资源
- 删除正在被使用的资源

### 幂等类
- 同一请求发送两次，结果应一致
- 重复提交表单

### 环境类
- 依赖服务不可用时的降级行为
- 网络超时
- 磁盘空间不足

## 应用步骤

1. 阅读接口文档和业务规则
2. 对照上述错误模式库逐条检查
3. 结合领域知识补充特定错误场景
4. 对每个推测编写测试用例

## 常见陷阱

- 过度依赖错误推测而忽略系统化技法
- 推测的场景没有可验证的预期结果
- 把"可能出错"当成"一定出错"，忽略了验证实际行为
