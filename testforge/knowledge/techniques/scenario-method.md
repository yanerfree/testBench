# 场景法（Scenario Method）

## 原理

从用户角度出发，模拟端到端的业务流程。包含一个基本流（happy path）和多个备选流（异常/分支路径）。

## 适用场景

- 多步骤业务流程（CRUD 全生命周期）
- 用户操作路径测试
- 集成测试 / E2E 测试
- 涉及多个接口协作的场景

## 应用步骤

1. 识别基本流（最常见的成功路径）
2. 识别备选流（每个步骤可能的分支）
3. 对每个场景编写完整的端到端用例

### 基本流模板

```
前置 → 步骤1 → 步骤2 → ... → 步骤N → 验证最终状态
```

### 备选流模板

```
前置 → 步骤1 → 步骤2（异常输入）→ 验证错误处理 → 恢复/终止
```

## 示例

RFC 连接器 MSHOST 基本流：
1. 登录获取 token
2. 填写 MSHOST 配置（host、systemName、clientNumber、port、userName、password）
3. 连通性测试 → 返回成功
4. 保存连接器（name、registerName 唯一）→ 返回 200 + 连接器 ID
5. 查询连接器列表 → 新建的连接器在列表中
6. 编辑连接器 → 修改 config 字段 → 保存成功
7. 删除连接器 → 返回成功
8. 查询列表 → 已删除的不再出现

备选流 1：步骤 3 连通失败
- 连通测试返回失败
- 尝试保存 → 应拒绝
- 修改配置 → 重新测试 → 成功后保存

备选流 2：步骤 4 name 冲突
- 保存时 name 已存在 → 返回 409
- 修改 name → 重新保存 → 成功

## 常见陷阱

- 只写基本流不写备选流
- 备选流只测试一个异常点，忽略从异常恢复的路径
- 忽略清理步骤（测试数据残留影响下次执行）
