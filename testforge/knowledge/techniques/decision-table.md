# 决策表（Decision Table）

## 原理

当多个条件组合影响系统行为时，用决策表穷举所有条件组合及对应的预期动作，确保不遗漏组合场景。

## 适用场景

- 多个布尔/枚举条件共同决定结果（如 connectType × openType × 是否通过连通测试）
- 权限组合（角色 × 操作 × 资源归属）
- 表单联动（字段 A 的值决定字段 B 是否必填）

## 应用步骤

1. 列出所有条件（输入变量）
2. 列出所有可能的动作（系统响应）
3. 构建决策表：
   | 条件1 | 条件2 | 条件3 | → 动作 |
   |-------|-------|-------|--------|
   | Y     | Y     | Y     | 允许   |
   | Y     | Y     | N     | 拒绝   |
   | ...   | ...   | ...   | ...    |
4. 合并等价规则（如果条件3不影响结果，合并两行）
5. 每条规则生成一条测试用例

## 示例

RFC 连接器创建：

| connectType | 连通测试通过 | openType | → 结果 |
|-------------|-------------|----------|--------|
| ASHOST      | 是          | public   | 创建成功，所有项目可见 |
| ASHOST      | 是          | project  | 创建成功，绑定指定项目 |
| ASHOST      | 否          | public   | 拒绝创建 |
| MSHOST      | 是          | public   | 创建成功 |
| MSHOST      | 否          | -        | 拒绝创建 |

## 常见陷阱

- 条件过多时组合爆炸 → 用正交表或 pairwise 方法裁剪
- 忽略"不适用"的组合（如 MSHOST 没有 systemNumber 字段）
- 只测正向组合不测异常组合
