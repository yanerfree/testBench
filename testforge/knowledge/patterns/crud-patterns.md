# CRUD 测试模式（CRUD Patterns）

## Create（创建）

| 场景 | 预期 |
|------|------|
| 全部必填字段正确 | 201，返回新资源（含 id） |
| 缺少必填字段 | 400，提示缺少哪个字段 |
| 唯一字段重复 | 409 Conflict |
| 关联的外键不存在 | 400 或 404 |
| 创建后立即查询 | 能查到刚创建的资源 |

## Read（查询）

| 场景 | 预期 |
|------|------|
| 按 ID 查询存在的资源 | 200 + 完整数据 |
| 按 ID 查询不存在的资源 | 404 |
| 列表查询（无过滤） | 200 + 分页数据 |
| 列表查询（带过滤条件） | 仅返回匹配项 |
| 分页：page=1, pageSize=1 | 只返回 1 条 |
| 分页：page 超过总页数 | 返回空列表，total 不变 |
| 排序 | 按指定字段升序/降序 |

## Update（更新）

| 场景 | 预期 |
|------|------|
| 正常更新可修改字段 | 200，返回更新后数据 |
| 更新不存在的资源 | 404 |
| 更新只读字段 | 被忽略或返回 400 |
| 更新唯一字段为已存在值 | 409 |
| 部分更新（PATCH 语义） | 只更新传入的字段 |
| 更新后再查询验证 | 数据确实已变更 |

## Delete（删除）

| 场景 | 预期 |
|------|------|
| 删除存在的资源 | 200 或 204 |
| 删除不存在的资源 | 404 |
| 删除后再查询 | 404 或不在列表中 |
| 删除被引用的资源 | 400（拒绝）或级联删除 |
| 软删除 vs 硬删除 | 软删除后 includeDeleted=true 可查到 |
| 重复删除 | 第二次返回 404（已删除） |

## 全生命周期场景

```
创建 → 查询确认存在 → 更新 → 查询确认变更 → 删除 → 查询确认不存在
```
