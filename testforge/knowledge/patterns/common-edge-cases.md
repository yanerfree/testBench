# 通用边界用例（Common Edge Cases）

## null / 空值系列

| 输入 | 预期行为 |
|------|---------|
| 字段缺失（不传该字段） | 必填字段返回 400；可选字段使用默认值 |
| 字段值为 null | 必填字段返回 400 |
| 字段值为空串 "" | 视业务而定，通常必填字段应拒绝 |
| 字段值为纯空格 "   " | 应被 trim 后视为空串，或拒绝 |
| 字段值为空对象 {} | 嵌套对象字段，内部必填缺失应返回 400 |
| 字段值为空数组 [] | 数组字段为空，视业务而定 |

## 注入攻击

| 输入 | 目的 |
|------|------|
| `'; DROP TABLE users; --` | SQL 注入 |
| `<script>alert(1)</script>` | XSS（存储型） |
| `{{7*7}}` | 模板注入（SSTI） |
| `../../etc/passwd` | 路径遍历 |
| `%00` | 空字节注入 |

预期：系统不崩溃、不执行注入代码、正确转义或拒绝输入。

## Unicode 和编码

| 输入 | 场景 |
|------|------|
| 中文字符 "测试连接器" | 多字节字符处理 |
| emoji "🔑🔐" | 4 字节 UTF-8 |
| 零宽字符 "​" | 不可见字符 |
| 混合 RTL "abc‮def" | 文本方向攻击 |
| 超长 Unicode（1000 个中文） | 字节长度 vs 字符长度 |

## 幂等性

- 同一 POST 请求发送两次：第二次应返回冲突（409）或幂等成功
- 同一 PUT 请求发送两次：结果应一致
- 同一 DELETE 请求发送两次：第二次应返回 404 或幂等成功

## 并发

- 两个请求同时创建同名资源：只有一个成功，另一个返回冲突
- 读取正在被更新的资源：返回更新前或更新后的完整数据，不应返回半更新状态
- 删除正在被引用的资源：应拒绝或级联处理
