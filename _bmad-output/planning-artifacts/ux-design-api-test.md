# UX Design Specification — testBench 接口测试模块

**Author:** Dreamer
**Date:** 2026-07-03

---

## 设计原则

- **对标 Apifox**：三栏布局（目录/步骤列表/请求编辑器），用户无学习成本
- **和平台统一**：复用 Ant Design 组件，目录树/表格/弹窗风格和用例管理页面一致
- **操作即保存**：编辑字段 blur 自动保存，不需要手动点保存按钮
- **状态可见**：场景状态、执行结果、变更提醒一眼可见

### 接口测试页面 — 两种视图

#### 视图一：列表视图（未选中场景时）

```
┌─ 目录树(220px) ─┬─ 工具栏 + 表格 ───────────────────────────┐
│ 测试场景    [+] │ [搜索🔍] [全部|草稿|已发布|已废弃]         │
│                 │                    [AI生成测试] [+新建场景] │
│ ▸ 用户管理 (8)  │                                           │
│   ▸ 创建 (3)    │ 场景ID    标题    来源  优先级 状态  操作   │
│   ▸ 更新 (3)    │ ─────────────────────────────────────── │
│   ▸ 删除 (2)    │ AT-0001  创建..  AI   P0    草稿  🗑    │
│ ▸ 认证 (4)      │ AT-0002  创建..  AI   P1    已发布 🗑    │
│   ▸ 登录 (2)    │ AT-0003  更新..  手动  P1    草稿  🗑    │
│   ▸ SSO (2)     │                                           │
│                 │                     共 8 条 < 1 >          │
└─────────────────┴───────────────────────────────────────────┘
```

**目录树**：
- 标题"测试场景" + 绿色"+"按钮（新建文件夹）
- 多级嵌套，展开箭头，每个节点显示场景计数
- hover 显示删除按钮（空文件夹才可删）
- 点击文件夹 → 右侧表格筛选该文件夹下的场景
- 默认选中第一个文件夹

**工具栏**：
- 搜索框（场景 ID 或标题）
- 状态筛选标签（全部/草稿/已发布/已废弃）
- AI 生成测试按钮（绿色主按钮）
- 新建场景按钮

**表格**：
- 列：场景 ID（monospace）、标题（加粗）、来源（AI/手动 Tag）、优先级（彩色 Tag）、状态（Tag）、操作（删除）
- 行可点击 → 进入场景详情（三栏视图）
- 支持批量选择（checkbox）→ 批量改状态/删除/移动

#### 视图二：场景详情（选中场景后 — 三栏）

```
┌─ 目录树 ─┬─ 步骤列表(300px) ──────┬─ 请求编辑器 ──────────────┐
│ 测试场景  │ AT-0001  AI  [草稿▾]  │ 创建用户-username缺失      │
│   [+]    │ 创建用户-参数校验      │                    [🗑][▶运行] │
│          │ 8 个步骤         [✕]  │                            │
│ ▸ 用户.. │                       │ [POST ▾] ${BASE_URL}/api/u │
│ ▸ 认证.. │ ● POST 登录-获取token  │                    [发送]  │
│          │ ○ POST 创建用户-user.. │                            │
│          │ ○ POST 创建用户-pass.. │ Body●  Headers 2  断言 2   │
│          │ ○ POST 创建用户-role.. │ 变量提取  响应              │
│          │ ○ POST 创建用户-user.. │                            │
│          │ ○ DELETE 清理用户1     │ JSON                 [保存] │
│          │ ○ DELETE 清理用户2     │ {                          │
│          │                       │   "role": "user",          │
│          │ [+ 添加步骤]          │   "password": "pwd123"     │
│          │                       │ }                          │
└──────────┴───────────────────────┴────────────────────────────┘
```

**步骤列表（中栏）**：
- 顶部：场景编号 + 来源标签(AI/手动) + 状态下拉 + 运行全部按钮 + ✕返回按钮
- 步骤列表：圆圈状态指示(○未执行/●通过/✕失败) + 方法标签(POST绿/DELETE红) + 步骤名
- 步骤名不截断，允许换行
- 底部：虚线"添加步骤"按钮
- 点击步骤 → 右栏显示请求详情
- 进入场景自动选中第一个步骤

**请求编辑器（右栏）**：
- 顶部：步骤名（可编辑 Input）+ 删除按钮 + 运行按钮（绿色）
- URL 栏：方法下拉(GET/POST/PUT/DELETE/PATCH) + URL 输入框（可编辑）+ 发送按钮
- Tab 栏：Body / Headers / 断言 / 变量提取 / 响应
  - Body：JSON 标签 + textarea 编辑器 + 保存按钮
  - Headers：表格（Key/Value 列），可增删改
  - 断言：表格（类型/字段/操作/期望值），可增删改
  - 变量提取：key-value 编辑
  - 响应：执行后显示（状态码 Tag + 耗时 + JSON 响应体）
- 所有字段 blur 自动保存

## 交互规范

### 状态颜色

| 状态 | 颜色 | Tag |
|------|------|-----|
| 草稿 | 默认灰 | `<Tag>草稿</Tag>` |
| 已发布 | 绿色 | `<Tag color="success">已发布</Tag>` |
| 已废弃 | 灰色 | `<Tag color="default">已废弃</Tag>` |

### 方法颜色

| Method | 颜色 |
|--------|------|
| GET | #4e8af0 蓝色 |
| POST | #52c41a 绿色 |
| PUT | #faad14 黄色 |
| DELETE | #e8453c 红色 |
| PATCH | #7c5cbf 紫色 |

### 优先级颜色

| Priority | Tag |
|----------|-----|
| P0 | `<Tag color="red">P0</Tag>` |
| P1 | `<Tag color="orange">P1</Tag>` |
| P2 | `<Tag color="blue">P2</Tag>` |
| P3 | `<Tag color="default">P3</Tag>` |

### 来源标签

| Source | Tag |
|--------|-----|
| AI 生成 | `<Tag color="blue">AI</Tag>` |
| 手动创建 | `<Tag>手动</Tag>` |

### 执行状态指示

| 状态 | 图标 | 说明 |
|------|------|------|
| 未执行 | ○ 灰色空心圆 | 从未运行过 |
| 通过 | ✓ 绿色实心 | `CheckCircleOutlined` |
| 失败 | ✕ 红色实心 | `CloseCircleOutlined` |
| 跳过 | — 灰色横线 | 步骤被禁用 |

### 编辑行为

| 字段 | 编辑方式 | 保存时机 |
|------|---------|---------|
| 步骤名 | 点击变 Input | blur 自动保存 |
| URL | 直接编辑 | blur 自动保存 |
| Method | 下拉选择 | 选择即保存 |
| Body | textarea | 点"保存"按钮 |
| Headers | 表格增删改 | blur 自动保存 |
| 断言 | 表格增删改 | blur 自动保存 |
| 状态 | 下拉选择 | 选择即保存 |

### 批量操作

选中多个场景后，工具栏出现批量操作：
- 批量改状态（草稿→已发布、已发布→已废弃）
- 批量删除
- 批量移动到文件夹

### 运行交互

**单步运行**：点步骤旁的"运行"按钮 → 按钮变 loading → 响应 Tab 显示结果
**批量运行**：
1. 选择环境下拉
2. 点"运行"按钮
3. SSE 流式显示进度（步骤名 + 通过/失败）
4. 完成后自动跳转到测试报告

### 响应式设计

- 最小宽度 1200px（三栏布局需要足够空间）
- 目录树可折叠（点击侧边栏图标收起）
- 步骤列表固定 300px，不可调整
- 请求编辑器占剩余空间
