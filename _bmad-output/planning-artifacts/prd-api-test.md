# Product Requirements Document - testBench 接口测试模块

**Author:** Dreamer
**Date:** 2026-07-03

## Executive Summary

testBench 接口测试模块为测试工程师和开发工程师提供 AI 驱动的接口测试自动化能力。用户只需提供 API 接口定义（从项目 API 列表选择、导入或手动输入），AI 即自动生成高质量、高覆盖率的测试场景和可执行步骤，选择运行环境后可直接执行，几乎不需要人工调整。

本模块解决的核心痛点：编写接口测试用例耗时（手动写每个请求参数和断言）、覆盖面不全（容易遗漏边界值和异常场景）、维护成本高（环境切换、token 管理、测试数据清理都需要手动处理）。

本模块是 testBench AI 测试生成能力的第一个试验场。接口测试结构化程度高、输入输出明确，适合先打磨 AI 生成质量和工作流程，为后续更复杂的功能测试场景（业务流程、多接口组合、状态机覆盖）积累经验和技术基础。

### What Makes This Special

**AI 生成质量是核心竞争力。** 与 Apifox/Postman 需要手动编写每个测试请求不同，testBench 基于接口参数 schema（字段类型、必填、长度、枚举、正则等约束）自动推导测试维度，生成覆盖必填校验、类型错误、边界值、安全测试的完整场景，目标是"生成即可用"。

**智能场景拆分。** AI 按测试维度自动拆分场景（正向验证、参数校验、安全测试、权限测试），正向和反向用例分离（边界值合法值归入正向、非法值归入反向），每个场景有单一明确的测试目的。

**智能目录管理。** 批量生成时 AI 根据接口 URL 路径自动识别模块（`/api/users` → 用户管理），自动创建文件夹并归类，不需要用户手动整理。

**生成后评审优化。** AI 生成的场景默认为草稿，进入评审优化阶段：
- **手动操作**：编辑步骤/断言/变量、复制场景、拆分场景、增删步骤
- **AI 优化**：用户提出建议 → AI 分析给出方案 → 用户确认后执行
- **状态流转**：草稿（可编辑、可调试执行）→ 已发布（不可编辑、正式执行进报告）→ 已废弃
- **批量操作**：勾选多个场景批量改状态、批量删除、批量移动文件夹

**测试数据零残留。** 环境级认证自动管理（多角色 token 缓存复用，不重复登录），正向测试创建的数据自动记录 ID，场景结束后自动 DELETE 清理。通过环境变量命名约定（`{ROLE}_USER` / `{ROLE}_PASS`）支持多角色切换，满足越权测试等多角色场景。

**双通道执行。** 平台内可视化操作（目录管理、步骤编辑、断言配置、一键运行）和 Claude Code CLI 两种方式，生成结果自动同步。

- **领域：** DevTools / 测试工具
- **复杂度：** 中等
- **项目上下文：** Brownfield — 在已有 testBench 平台上新增模块，复用已有的环境管理、用户认证、API 接口管理等基础设施

## Success Criteria

### User Success

- **生成即可用**：AI 生成的测试场景，用户不修改直接发布的比例 ≥ 90%
- **覆盖维度完整**：每个接口自动覆盖正向测试（含边界合法值）、参数校验（含边界非法值）、安全测试、权限测试维度
- **零脏数据**：正向测试创建的数据，场景结束后 100% 自动清理
- **环境无感切换**：选择环境后 token 自动管理、多角色自动切换，用户不需要手动处理认证

### Business Success

- **为功能测试场景积累经验**：接口测试的 AI 生成质量和流程模式可复制到后续业务场景测试
- **替代手动编写**：相比手动在 Apifox/Postman 写测试，效率提升 10 倍以上

### Technical Success

- **单步执行延迟**：单个请求步骤执行 < 500ms（网络延迟除外）
- **生成速度**：单接口场景生成 < 30 秒，批量按接口数线性增长
- **导出可执行**：导出的 pytest 项目开箱即用，`pip install && pytest` 直接跑

### Measurable Outcomes

- 生成后直接发布率 ≥ 90%
- 测试数据清理成功率 100%
- 导出脚本一次执行通过率 ≥ 95%

## Product Scope

### MVP

**生成（双通道）：**
- **Claude Code 通道（主力）**：读项目源码 → 解析路由/参数/校验规则 → 通过 MCP 工具生成 → 自动同步到平台
- **平台内通道（辅助）**：从项目 API 列表选择 / 导入 / 手动输入接口定义 → AI 生成
- 生成时选择目标文件夹，批量生成时 AI 按 URL 路径自动创建目录归类
- 智能场景拆分：按测试维度（正向/参数校验/安全/权限），正反分离，边界值分别归入正向（合法）和反向（非法）

**管理：**
- 独立文件夹目录树（CRUD、多级嵌套）
- 场景列表（搜索、筛选、分页）
- 状态流转：草稿（可编辑、可调试执行）→ 已发布（不可编辑、正式执行进报告）→ 已废弃
- 批量操作：批量改状态、批量删除、批量移动文件夹
- 来源标记：AI 生成 / 手动创建

**评审优化：**
- 手动操作：编辑步骤/断言/变量、复制场景、拆分场景、增删步骤
- AI 优化：用户提出建议 → AI 分析给出方案 → 用户确认后执行

**执行：**
- 环境级 token 自动管理（登录一次缓存复用，过期自动刷新）
- 多角色支持（环境变量 `{ROLE}_USER` / `{ROLE}_PASS`，越权测试场景切换角色）
- 正向测试创建的数据自动记录 ID，场景结束后自动 DELETE 清理
- 单步执行、批量执行（选中的场景或整个文件夹）
- 断言验证（HTTP 状态码、响应字段、响应包含文本）

**测试报告：**
- 每次执行自动生成报告，与已有测试报告页面集成
- 报告类型字段：接口测试 / 功能场景测试（预留）
- 报告命名规则：单场景=场景名称，同文件夹批量=文件夹名+时间，跨文件夹批量=接口测试回归+时间
- 报告详情可下钻到每个步骤的请求/响应/断言结果

### Growth Features（Post-MVP）

- 导出完整 pytest 项目（选择场景导出弹窗：左侧目录树+右侧列表，导出 zip 开箱即用）
- 从 API 接口页面一键生成
- 脚本视图（在线查看 pytest 代码）

### Vision（Future）

- 业务场景测试（多接口组合、状态机、工作流）
- CI/CD 集成（定时执行、流水线触发）
- 智能回归（代码变更自动识别影响的测试场景）

## User Journeys

### 旅程一：全栈工程师 — 通过 Claude Code 生成测试（主力场景）

**小李**，全栈工程师，刚写完用户管理模块的 CRUD 接口，需要补接口测试。

**首次接入**：
1. 打开 testBench 平台，进入项目 → 侧边栏「AI 智能」→「MCP 工具」页面
2. 页面显示 MCP Server 地址和 API Key，点击「生成 Claude Code 配置」
3. 页面展示操作步骤：MCP 配置片段（复制合并到项目 `.mcp.json`）+ Skill 安装（一行命令自动下载或手动下载 ZIP）
4. 小李按步骤操作，在项目目录启动 Claude Code

**日常使用**：
5. 输入："为 /api/users 的所有接口生成接口测试"
6. Claude Code 读取项目源码，解析出 5 个接口的路由、参数、校验规则
7. 通过 MCP 工具调用 testBench 平台生成 15 个测试场景（按维度拆分：正向/参数校验/安全/权限）
8. AI 自动识别模块，创建「用户管理」文件夹，场景自动归入
9. 小李打开 testBench 平台接口测试页面，场景已在目录中
10. 快速浏览，质量不错，批量改状态为「已发布」
11. 选择 development 环境，批量执行
12. 2 个场景失败——发现是接口确实有 bug（密码长度校验没做）
13. 修完代码重跑，全部通过，测试报告自动生成

**关键时刻**：从写完代码到测试通过，10 分钟搞定。

### 旅程二：全栈工程师 — 平台内生成测试

**小王** 不用 Claude Code，直接在平台操作。

1. 打开接口测试页面，点「AI 生成测试」
2. 选择输入方式：从项目 API 列表勾选「用户管理」下的 5 个接口（也可手动粘贴接口定义）
3. 选择目标文件夹（不选则 AI 自动按模块创建）
4. 点「开始生成」，SSE 流式显示进度
5. 生成完成，15 个场景出现在目录树中，按维度拆分（正向/参数校验/安全/权限）
6. 草稿状态，先调试运行几个确认没问题
7. 批量改状态为「已发布」

### 旅程三：全栈工程师 — 评审优化

**小王** 发现 AI 生成的某个场景需要调整。

**手动优化**：
1. 打开场景，直接编辑步骤名/URL/Body/断言
2. 复制一个场景作为新场景修改
3. 把某个场景里的几个步骤拆分成独立场景

**AI 优化**：
4. 输入修改意见："增加 username 包含中文字符的测试"
5. AI 分析后给出方案：新增一个步骤，展示具体的 URL/Body/断言
6. 小王确认，AI 执行修改
7. 调试运行通过，改状态为已发布

### 旅程四：全栈工程师 — 回归测试

**小李** 改了认证模块代码，需要确认没有影响其他接口。

1. 打开接口测试页面，选中所有已发布场景
2. 选择 development 环境，批量执行
3. 报告自动生成：「接口测试回归 2026-07-03 14:30」
4. 通过 48/50，2 个失败
5. 点进失败场景，下钻到具体步骤的请求/响应/断言结果
6. 定位问题，修复代码，重跑通过

### 旅程五：Admin — 查看报告

**张主管** 查看接口测试覆盖情况。

1. 打开测试报告页面，筛选类型为「接口测试」
2. 看到最近一次全量回归：通过率 97.5%
3. 下钻到失败步骤查看详情
4. 只读操作，不编辑场景

### 已知技术问题

**MCP Mock 与 MCP Server 冲突**：MCP Mock 和真实 MCP Server 共用同一个地址 `/mcp/`，通过 `mock_enabled()` 开关切换。Mock 开启后所有外部 Claude Code 调用也会拿到假数据。需要在架构设计阶段解耦，确保互不影响。

## Functional Requirements

### 1. AI 测试生成

- **FR1**: 用户可以通过 Claude Code 输入自然语言描述，AI 读取项目源码自动生成接口测试场景
- **FR2**: 用户可以在平台内从项目 API 列表选择接口，AI 生成对应的测试场景
- **FR3**: 用户可以在平台内手动粘贴接口定义文本，AI 生成测试场景
- **FR4**: AI 生成时自动按测试维度拆分场景（正向测试、参数校验、安全测试、权限测试）
- **FR5**: AI 生成时正向和反向用例分离，边界合法值归入正向场景、非法值归入反向场景
- **FR6**: 批量生成时 AI 根据接口 URL 路径自动识别模块并创建文件夹归类
- **FR7**: 每个生成的步骤包含具体的请求参数值和断言（非笼统描述）

### 2. 场景管理

- **FR8**: 用户可以创建、查看、编辑、删除测试场景
- **FR9**: 用户可以创建多级文件夹目录组织场景
- **FR10**: 用户可以在文件夹之间移动场景
- **FR11**: 场景支持状态流转：草稿→已发布→已废弃
- **FR12**: 草稿状态可编辑、可调试执行；已发布不可编辑、正式执行进报告；已废弃不可执行
- **FR13**: 用户可以批量选择场景执行改状态、删除、移动操作
- **FR14**: 场景自动标记来源（AI 生成 / 手动创建）
- **FR15**: 用户可以搜索场景（按 ID 或标题）和按状态筛选

### 3. 步骤编辑

- **FR16**: 用户可以在场景内添加、编辑、删除、排序步骤
- **FR17**: 每个步骤可编辑：名称、HTTP 方法、URL、Headers、Body、断言、变量提取
- **FR18**: 用户可以复制整个场景为新场景
- **FR19**: 用户可以选中多个步骤拆分为独立场景

### 4. AI 优化

- **FR20**: 用户可以对已有场景输入修改意见，AI 分析后给出优化方案
- **FR21**: AI 优化方案需用户确认后才执行修改
- **FR22**: AI 优化可以增加步骤、减少步骤、修改断言、调整变量

### 5. 测试执行

- **FR23**: 用户可以执行单个步骤并查看请求/响应/断言结果
- **FR24**: 用户可以批量执行整个场景或选中的多个场景
- **FR25**: 执行时自动根据环境配置获取 token 并缓存复用
- **FR26**: 执行时支持多角色切换（通过环境变量 `{ROLE}_USER`/`{ROLE}_PASS`）
- **FR27**: 正向测试创建的数据自动记录 ID，场景结束后自动清理
- **FR28**: 步骤间支持变量传递（前一步提取的值传给后续步骤）

### 6. 测试报告

- **FR29**: 每次执行（单场景或批量）自动生成测试报告
- **FR30**: 报告与已有测试报告页面集成，支持按类型筛选（接口测试/功能场景测试）
- **FR31**: 报告名称自动生成：单场景=场景名，同文件夹批量=文件夹名+时间，跨文件夹=接口测试回归+时间
- **FR32**: 报告可下钻到每个步骤的请求/响应/断言详情

### 7. Claude Code 集成

- **FR33**: 平台提供 MCP Server 供外部 Claude Code 连接（真实数据，非 Mock）
- **FR34**: 平台提供 API Key 生成/吊销功能用于 MCP 认证
- **FR35**: 平台在 MCP 工具页面提供 Claude Code 连接配置（MCP 配置片段 + Skill 安装方式）
- **FR36**: Skill 文件支持一行命令自动安装或手动下载 ZIP
- **FR37**: MCP Mock 与真实 MCP Server 解耦，互不影响

## Non-Functional Requirements

### 性能

- **NFR1**: 单步执行延迟 < 500ms（不含被测系统网络延迟）
- **NFR2**: 单接口场景 AI 生成 < 30 秒
- **NFR3**: 页面加载和操作响应 < 1 秒
- **NFR4**: 批量执行 100 个步骤不超过 3 分钟（串行执行）

### 安全

- **NFR5**: MCP Server API Key 使用 SHA256 哈希存储，传输使用 Bearer Token
- **NFR6**: 环境变量中的密码字段不在前端明文显示（可选显示）
- **NFR7**: 测试报告中脱敏处理请求中的密码/Token 字段

### 可靠性

- **NFR8**: 测试数据清理成功率 100%（清理失败需记录日志并告警）
- **NFR9**: AI 生成失败时已完成的部分不丢失（断点续生成或保留已生成场景）
- **NFR10**: 批量执行中单步失败不影响后续步骤执行（继续执行，最后汇总报告）

### 兼容性

- **NFR11**: 支持 Chrome/Edge/Firefox 最新版
- **NFR12**: MCP Server 兼容 MCP 2025-03-26 规范（StreamableHTTP）
